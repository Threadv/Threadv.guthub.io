session

客户端每次和服务器交互时不需要每次都传回所有的cookie值，而是只要传回一个ID，这个ID是客户端第一次访问服务器时生成的，而且每个客户端是唯一的。

  • SESSION 工作
    服务端可以通过session ID 创建HttpSession对象，第一次触发通过reques.getSession()方法。如果当前的session ID没有对应的HttpSession对象，就创建一个新的，并且将这个对象加到org.apach.catalina.Manager的sessions容器中保存。Manager类将管理所有的Session生命周期，Session过期将被回收，服务器关闭，Session将被序列化到磁盘。只要HttpSession对象存在，用户就可以根据Session ID来获取这个对象，做到对状态的保持。

从request.getSession中获取的HttpSession对象实际是StandardSession对象的门面对象，这与前面的Request和Servlet是一样的原理，从Request中获取的Session对象保存在org.apache.catalina.Manager类中，它的实现类是org,apache.catalina.session.StandardManager,通过requestedSessionId从StandardManager的sessions集合中取出的StandarSession对象。由于一个requestedSessioId对应一个访问的客户端，多以一个客户端也就对应一个StandardSession对象，这个对象正是保存我们创建的的Seesion值的。

StandardManager类负责Servlet容器中所有的的StandardSession对象的生命周期管理。当Servlet容器长期或者关闭时，StandManager负责持久化没有过期的StandardSession对象，它会将所有的StandardSession对象持久化到一个以“SESSIONS.ser”为文件名的文件中，到Servlet容器重启时，也就是StandarManager初始化时，它会重新读取这个文件，解析出所有session对象，重新保存在StandardManager的sessions集合中。
